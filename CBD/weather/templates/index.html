Se desea construir una pagina web, que permita buscar obras de arte para la decoración del hogar. Además se recomendará obras a usuarios en función de los gustos de éstos.

Almacenar la información siguiente:

  1. Categoria: IdCategoria y Nombre.
  2. Usuario: IdUsuario y Nombre.
  3. Obra: IdObra, Titulo, Descripcion, Existencias, UrlImagen, Precio, Categorias y Puntuaciones.
  4. Puntuacion: IdUsuario, IdObra, Puntuacion, Fecha y Descripcion.

Extrayendo los datos de la página web: https://www.decorarconarte.com

Se pide un programa con un menú principal, que permita las siguientes opciones:

  1. Listado de todos los usuarios que han puntuado al menos 1 obra
  2. Listado con las 12 obras mejor puntuadas en orden decreciente. Para el calculo de la puntuacion se ha hecho la media de todas las puntuaciones de la obra.
  3. Dada una categoría escogida mediante un selector, muestre todas las obras de esa categoría.
  4. Dada una obra escogida mediante un selector (solo entre las puntuadas), muestre todas sus puntuaciones.
  5. Dado un usuario escogido (entre los que han puntuado al menos una obra), le recomiende obras que no haya puntuado, usando un sistema de recomendación Basado en Contenido.
  6. Dado un precio en formato inglés (punto como separador de decimales) y sin comas, le muestre obras con un precio por debajo del dado usando un esquema Whoosh.

Para el esquema Whoosh hace falta poblar primero la Base de Datos

Para el sistema de recomendación primero es necesario definir las preferencias del usuario y las características de las obras:

  • Para establecer las características de las obras establecer las diez categorías más frecuentes de cada obra y almacenarlas.
  • Para determinar las preferencias de un usuario, tomar las cinco obras mejor puntuadas, y obtener las diez categorías más frecuentes de esas obras. Esas categorías definen el perfil del usuario.
  • Comparar el perfil del usuario con las características de las obras y devolver los más parecidos (p.e. usando el coeficiente de Dice).
